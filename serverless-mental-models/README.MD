# Serverless Mental Models | AWS Lambda Course for Beginners

Most serverless tutorials teach you how to deploy a Lambda function. This series teaches you how to *think* about serverless — so you can make good architecture decisions instead of just following steps.

I've given talks on serverless at AWS Summit and AWS re:Invent. This series is the version I wish had existed when I was learning: honest, practical, and beginner-friendly without being dumbed down.

## What's in this series

**▶ Video 1 — [When Serverless Is the Wrong Choice](https://youtu.be/uq1LUJWobYg)**
The honest picture first. Cold starts, the 15-minute ceiling, real cost math from the AWS Calculator, and stateful workloads that fight the model. Includes a decision framework for knowing when to use serverless and when to reach for something else.

**▶ Video 2 — Stateless Does Not Mean Simple** *(Coming Soon)*
Your functions are stateless. Your system is not. This video maps out exactly where the complexity goes and how to manage it.

**▶ Video 3 — Why Serverless Is About Failure Domains** *(Coming Soon)*
The real reason to use serverless isn't cost or scaling. It's failure isolation. Once you understand blast radius and failure domains, every architecture decision looks different.

**▶ Video 4 — Build a Minimal Serverless API (hands-on)** *(Coming Soon)*
We build a real REST API using API Gateway, Lambda, and DynamoDB. Every design decision connects back to the mental models from the first three videos.
